# Node.js Backend Architecture (Express + MySQL + Sequelize)

## 1. Directory Structure
The backend must follow a strict modular, separation-of-concerns architecture. All business logic must be kept out of the routing files, and all database interactions must go through Sequelize.

```text
/backend
├── /config
│   └── database.js           # Sequelize instance and MySQL connection setup
├── /middleware
│   └── authMiddleware.js     # JWT verification and role-based access control
├── /models
│   ├── index.js              # Initializes Sequelize and defines model associations (relationships)
│   ├── patient.js            # Sequelize model definition for Patients
│   ├── therapist.js          # Sequelize model definition for Therapists
│   └── ...                   # (One Sequelize model file per database table)
├── /controllers
│   ├── patientController.js  # Business logic and req/res handling for Patients
│   ├── therapistController.js# Business logic and req/res handling for Therapists
│   └── ...                   # (One controller file per entity)
├── /routes
│   ├── patientRoutes.js      # Express router mapping endpoints to Patient controllers
│   ├── therapistRoutes.js    # Express router mapping endpoints to Therapist controllers
│   └── ...                   # (One route file per entity)
├── app.js                    # Main Express application (Registers routes and middleware)
├── server.js                 # Entry point (Starts the server/nodemon and syncs Sequelize)
└── package.json

DONT SKIP Folder Communication logic: Client Request -> app.js -> routes -> controllers -> models -> MySQL
