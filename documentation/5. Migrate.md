# PHMS — Migration Phases

## ⚠️ CRITICAL INSTRUCTIONS — READ FIRST
1. **Each phase creates ONLY two things**: Model files + **ONE single migration file**. **Nothing else.**
2. **DO NOT** create controllers, routes, middleware, or register anything in `app.js`. Those come later in the CRUD phase.
3. **Model files** go in: `backend/models/` — one file per table (e.g. `models/Patient.js`)
4. **ONE migration file per phase** goes in: `backend/migrations/` — it creates ALL tables for that phase inside a single `up()` method.
5. After creating all model files for your phase, **register them in `models/index.js`** (import + add to `db` object).
6. **`timestamps: false`** for EVERY model and EVERY table in EVERY phase. **DO NOT** add `createdAt` or `updatedAt` columns. **DO NOT** set `timestamps: true`.
7. After completing the phase, **OUTPUT THIS EXACT COMMAND** to the user so they can run it in the terminal:
   ```
   npx sequelize-cli db:migrate --config config/database.js
   ```
   Tell the user: "Run this in your terminal to apply the migration."

**Total files per phase**: 6 or 5 model files + 1 migration file.
**Total migrations across entire project**: 3 files.

## Tech Stack
- **ORM**: Sequelize + sequelize-cli
- **DB**: MySQL, database name `phms`
- **Models**: OOP class-based (`class extends Model`, `Model.init()`, factory export)
- **Architecture**: `config/database.js` → `models/index.js` → individual model files in `models/` + migration files in `migrations/`

## Model Pattern (ALL phases follow this — `timestamps: false` ALWAYS)
```javascript
const { DataTypes, Model } = require('sequelize');

class EntityName extends Model {
  static associate(models) {
    // belongsTo / hasMany / hasOne defined here
  }
}

module.exports = (sequelize) => {
  EntityName.init(
    {
      // columns here — NO createdAt/updatedAt
    },
    {
      sequelize,
      tableName: 'ExactTableName',
      timestamps: false,  // MUST be false for every model
    }
  );
  return EntityName;
};
```

## Migration Pattern (ALL phases follow this)
Each phase = **ONE migration file** with multiple `createTable` calls inside `up()`, and matching `dropTable` calls in `down()` (in reverse order). **NO `createdAt` or `updatedAt` columns.**
```javascript
'use strict';
module.exports = {
  async up(queryInterface, Sequelize) {
    // Create tables IN ORDER (parent tables first, then child tables with FKs)
    await queryInterface.createTable('ParentTable', {
      id: { type: Sequelize.INTEGER, primaryKey: true, autoIncrement: true },
      // ... only the columns from the spec, NO timestamps
    });
    await queryInterface.createTable('ChildTable', { /* columns + FK refs, NO timestamps */ });
    // ... all tables for this phase
  },
  async down(queryInterface) {
    // Drop in REVERSE order (children first, then parents)
    await queryInterface.dropTable('ChildTable');
    await queryInterface.dropTable('ParentTable');
  },
};
```

## After EACH phase, register new models in `models/index.js`:
```javascript
const NewModel = require('./NewModel')(sequelize);
// add to db object, then the associate loop handles the rest
```

## Migration Command (output at end of EVERY phase)
```bash
npx sequelize-cli db:migrate --config config/database.js
```

# PHASE 1 — Identity System (6 tables, no FK dependencies)
**Create**: 6 model files in `models/` + **1 migration file** in `migrations/`
**Migration file**: `migrations/20250101000001-phase1-identity-system.js`
**DO NOT create**: controllers, routes, or any other files
**`timestamps: false` on every model and every `createTable` call**

**Drop order in `down()`**: RefreshTokens → UserTokens → UserClaims → UserRoles → Roles → Users

## Quick Reference Commands
```bash
# Run pending migrations (use this one)
npx sequelize-cli db:migrate --config config/database.js

# Check status
npx sequelize-cli db:migrate:status --config config/database.js

# Undo last migration
npx sequelize-cli db:migrate:undo --config config/database.js

# Undo ALL migrations
npx sequelize-cli db:migrate:undo:all --config config/database.js
```
