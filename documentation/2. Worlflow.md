# Patient Workflow: Step-by-Step

## 1. Authentication (Login)
* **Action:** Patient enters email and password.
* **System Response:** Validates credentials via the Users table, issues a JWT token, and logs the session in RefreshTokens.

## 2. Dashboard Initialization (Overview)
* **Action:** Patient lands on the home screen.
* **System Response:** Uses the patient's ID to fetch and display:
    * Personal info from Patients (Editable).
    * Upcoming schedule from Appointments (Read-only).
    * Active goals from TreatmentPlans (Read-only).
    * Pending payments from Invoices (Read-only).

## 3. Booking an Appointment
* **Action:** Patient clicks "Request Appointment".
* **System Response:** Fetches available doctors from Therapists.
* **Action:** Patient selects a therapist, date, and time.
* **System Response:** Creates a new record in Appointments with a "Pending" status.

## 4. Pre-Session Assessment (Questionnaires)
* **Action:** Patient opens a pending assessment assigned by the clinic.
* **System Response:** Loads the blank JSON template from Questionnaires.
* **Action:** Patient fills it out and clicks "Submit".
* **System Response:** Saves the answers into QuestionnaireResponses, linked to the patient's ID.

## 5. Post-Session Review (Follow-up)
* **Action:** After a session happens in real life, the patient logs back in to check updates.
* **System Response:** * Displays new medical condition (if given) from Diagnoses.
    * Displays new homework/goals from TreatmentPlans.
    * Displays a new bill to pay from Invoices.
    * Shows basic session history (date/time) from Sessions. (Crucial Note for Docs: SessionNotes are strictly hidden from this view).

# Therapist Workflow: Step-by-Step

## 1. Authentication (Login)
* **Action:** Therapist enters email and password.
* **System Response:** Validates credentials via the Users table, issues a JWT token, and logs the session in RefreshTokens.

## 2. Dashboard Initialization (Daily Overview)
* **Action:** Therapist lands on their home screen.
* **System Response:** Uses the therapist's ID to fetch and display:
    * Professional profile from Therapists.
    * Today's schedule and pending requests from Appointments.
    * Assigned physical locations from Rooms (based on confirmed appointments).
    * Pre-session test results from QuestionnaireResponses submitted by their patients.

## 3. Appointment Management
* **Action:** Therapist reviews a "Pending" appointment request from a patient and clicks "Accept".
* **System Response:** Updates that specific record in the Appointments table, changing the statusi to "Confirmed".

## 4. Conducting the Session
* **Action:** The patient arrives, and the Therapist clicks "Start Session" on the system.
* **System Response:** Inserts a brand new record into the Sessions table. This permanently links the pacienti_id and terapisti_id together for this specific date and logs the exact ora_fillimit (start time).
* **Action:** The meeting ends, Therapist clicks "End Session".
* **System Response:** Updates the Sessions record with the ora_mbarimit (end time).

## 5. Clinical Documentation (Post-Session Paperwork)
* **Action:** Therapist writes down private clinical observations.
* **System Response:** Inserts a new record into SessionNotes (Strictly linked 1-to-1 with the seanca_id that just ended. Note for Docs: This table is never exposed to the patient).
* **Action:** Therapist formally diagnoses the patient's condition.
* **System Response:** Inserts a new record into Diagnoses, linking the condition to the patient.
* **Action:** Therapist assigns homework or recovery goals based on the diagnosis.
* **System Response:** Inserts a new record into TreatmentPlans, linking it to the newly created diagnoza_id and the patient.
