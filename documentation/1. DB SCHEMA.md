# Database Schema: Psychological Clinic Management System

## 1. Identity System Tables (Mandatory)
These tables manage authentication and authorization.
* **Users**: (Id, Email, PasswordHash)
* **Roles**: (Id, Name)
* **UserRoles**: (UserId, RoleId)
* **UserClaims**: (Id, UserId, ClaimType, ClaimValue)
* **UserTokens**: (UserId, LoginProvider, Name, Value)
* **RefreshTokens**: (Id, Token, Expires, Created, Revoked, UserId)

## 2. Core Domain Entities
* **Patients**: (id, emri, mbiemri, email, telefoni, data_lindjes, gjinia, adresa, kontakti_urgjences, data_regjistrimit, UserId_FK)
* **Therapists**: (id, emri, mbiemri, email, telefoni, specializimi, nr_licences, kualifikimet, data_punesimit, biografia, UserId_FK)
* **Rooms**: (id, emertimi, kati, lloji, kapaciteti, pajisjet, statusi)
* **Questionnaires**: (id, emertimi, pershkrimi, lloji, pyetjet_json, data_krijimit)

## 3. Interaction & Clinical Entities
* **Appointments**: (id, pacienti_id, terapisti_id, room_id, data_takimit, ora_takimit, kohezgjatja_min, lloji, statusi, arsyeja_anulimit)
  * *Note on `room_id`*: This is a **Nullable** Foreign Key. It is NOT selected by the patient during booking. It is assigned later by the Admin/Therapist before the session starts.
* **Sessions**: (id, pacienti_id, terapisti_id, data_seances, ora_fillimit, ora_mbarimit, lloji_seances, statusi, shenime_private, shenime_pacienti)
* **Diagnoses**: (id, pacienti_id, terapisti_id, kodi_diagnoz, emertimi, pershkrimi, data_diagnostikimit, ashpersia)
* **TreatmentPlans**: (id, pacienti_id, terapisti_id, diagnoza_id, emertimi, objektivat, data_fillimit, data_perfundimit, statusi)
* **SessionNotes**: (id, seanca_id, shenime, progresi, detyrat_shtepise, plani_ardhshem, data_krijimit)
* **QuestionnaireResponses**: (id, pyetesori_id, pacienti_id, pergjigjjet_json, skori_total, data_plotesimit)
* **Invoices**: (id, pacienti_id, seanca_id, shuma, zbritja, shuma_perfundimtare, data_fatures, statusi_pageses)

## 4. Entity Relationships
* **Appointments -> Patients (Many-to-One)**: Cdo takim behet per nje pacient.
* **Appointments -> Therapists (Many-to-One)**: Cdo takim caktohet me nje terapist.
* **Appointments -> Rooms (Many-to-One, Nullable)**: Cdo takim i konfirmuar zhvillohet ne nje dhome.
* **Sessions -> Patients (Many-to-One)**: Cdo seance eshte per nje pacient.
* **Sessions -> Therapists (Many-to-One)**: Cdo seance drejtohet nga nje terapist.
* **SessionNotes -> Sessions (One-to-One)**: Cdo shenim lidhet me nje seance.
* **Diagnoses -> Patients (Many-to-One)**: Cdo diagnoze i perket nje pacienti.
* **Diagnoses -> Therapists (Many-to-One)**: Cdo diagnoze vendoset nga nje terapist.
* **TreatmentPlans -> Patients (Many-to-One)**: Cdo plan trajtimi eshte per nje pacient.
* **TreatmentPlans -> Diagnoses (Many-to-One)**: Cdo plan lidhet me nje diagnoze.
* **TreatmentPlans -> Therapists (Many-to-One)**: Cdo plan trajtimi krijohet nga nje terapist.
* **Invoices -> Patients (Many-to-One)**: Cdo fature i drejtohet nje pacienti.
* **Invoices -> Sessions (Many-to-One)**: Cdo fature lidhet me nje seance.
* **QuestionnaireResponses -> Patients (Many-to-One)**: Cdo pergjigje jepet nga nje pacient.
* **QuestionnaireResponses -> Questionnaires (Many-to-One)**: Cdo pergjigje bazohet ne nje shabllon pyetesori.

## 5. Required CRUD Operations (Minimum)
* CRUD per Perdoruesit e Sistemit (Identity)
* CRUD per Pacientet
* CRUD per Terapistet
* CRUD per Seancat Terapeutike
* CRUD per Diagnozat
* CRUD per Planet e Trajtimit
* CRUD per Shenimet e Seancave
* CRUD per Takimet
* CRUD per Faturat
* CRUD per Dhomat/Sallat
* CRUD per Pyetesoret
* CRUD per Pergjigjjet e Pyetesoreve
